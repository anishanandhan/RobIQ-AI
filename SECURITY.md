# Security Policy

## Supported Versions

We actively maintain and support the following versions of the RobIQ-AI Candidate Ranker:

| Version | Supported |
| :--- | :--- |
| v1.x | Yes |
| < v1.0 | No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it to us as soon as possible. We take all security issues seriously and appreciate your help in keeping this project secure.

Please do **NOT** open a public issue on GitHub for security vulnerabilities. Instead, report them directly by emailing:

**security@robiq-ai.com** (or directly to primary contact: **anishanandhan.al2022@vitstudent.ac.in**)

When reporting, please include:
1. A detailed description of the vulnerability and its potential impact.
2. Step-by-step instructions (or a proof-of-concept script) to reproduce the issue.
3. Information about your environment.

### Our Commitment

* **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
* **Triage**: We will investigate and verify the vulnerability.
* **Remediation**: We target a fix or patch within 14 days of verification.
* **Coordinated Disclosure**: We ask that you do not publicly disclose the vulnerability until we have patched it. We commit to crediting you for the discovery upon release of the fix.

## Out of Scope

The following types of testing are strictly out of scope for this security policy:
* Social engineering or phishing attacks against project developers.
* Physical security attacks against developer workstations or servers.
* Distributed Denial of Service (DDoS) attacks against any hosted sandbox or demo environments.
* Security vulnerabilities in third-party services, cloud providers, or host platforms (e.g. GitHub, Vercel, HuggingFace).
